How UI/UX designs create meaning

Overall website

In developing the website's UI/UX, I focused on creating an engaging atmosphere that aligns with the website’s space exploration theme through consistent use of colors, dynamic animations, navigational elements (nav, footer, breadcrumbs, side nav, back to top button, next and previous buttons) and strategic placement of text and images. By reducing text and prioritizing visual storytelling, I made it easier for users to grasp the context at a glance, reducing the cognitive load and enhancing accessibility. This applied for the timeline. This approach is supported by clear headings and subheadings, which help users navigate the site more efficiently and familiarize themselves with its structure quickly. I also added buttons that would take the user to more information if they were more interested in the particular topic that I briefly touch on as to keep the focus on the visuals and “viewing the space atmosphere”. The placement of text and images was strategically used to guide the users eye flow through the page and distract them from being engaging and overwhelmed with to much text, as seem on the home and about page, where the images appear along side the text and switch for every section to give the user a quick break from the text and remind them of space and planets , to create more wonder and thoughts.

By minimizing text and using icons and symbols, taking advantages of iconic memory,I have crafted a user experience that encourages visual exploration and interaction, which is far more engaging than static text-based content. This decision promotes an intuitive journey through the site, where discovery is driven by the user’s curiosity and visual cues, aligning with the goal of making the site a visually-driven exploration of space. Subtle borders and lines act as visual cues that organize information and divide content into digestible sections. These elements, alongside other dynamic animations, ensure the interface remains lively and engaging without overwhelming the user. Animations like the stars on the homepage and how content appears and behaves with subtle movement serve to bring the site to life, reinforcing the theme of liveliness and constant activity which is crucial for keeping the user interested.

The glow effects on headings and hover states, designed to mimic the glowing of stars, enhance the thematic consistency of exploring space, making the user experience feel more immersive. The choice of a color palette with blues, greys, and blacks creates an atmosphere reminiscent of space, while selective use of yellow highlights interactive elements, guiding users to understand that these components are interactive and impactful, such as buttons that alter the state of the site or activate features.

The design of interactive elements based on conventional usability principles allows users to navigate the interface effortlessly without needing detailed instructions (like the standard placement of the navigation, control buttons for next, previous, top, pause, resume, zoom, refresh …). This ease of interaction, not needing instructions, is critical for engaging users effectively, allowing them to explore content deeper without frustration. I considered adding tooltips for clarity, but to avoid the annoyance that persistent tooltips can cause, I limited their use to essential interactions only. To further aid navigation, I incorporated clear feedback mechanisms for different interaction states and provided additional navigational tools like sticky side navigation, breadcrumbs, and 'back to top' buttons, which enhance user flow and reduce the effort required to navigate through dense information.

As users interact with the website, they uncover layers of information about the universe, mimicking the act of exploration and discovery. The interactive tools such as zoom, pan, filters, and pause/resume buttons empower users to control their exploration, adjusting the pace and focus of their learning experience. Challenges

UI Alignment and Flexbox Usage:

One of the most significant challenges encountered during the development was aligning interactive UI elements in a way that was intuitive from a design perspective, while also implementing it effectively through code. Specifically, the use of CSS Flexbox for positioning containers that included data visualizations alongside interactive controls proved particularly troublesome. This complexity was compounded by the responsiveness required, leading to occasional overlapping of elements that did not align with the intended design.

Integration of Live API Data:

Another pressing issue was the integration of live API data, particularly for the exoplanet interactive map. The initial implementation faced a synchronization problem where the search bar and the map markers could not display simultaneously. If the search bar was populated with planet names, the corresponding markers on the map would disappear, and vice versa. This significantly impacted the usability and functionality of the interactive map, a key feature of the website. To resolve this, I tried a global variable to manage the state of the displayed data across different UI components. By centralizing the data state, both the search bar and the map could react to changes in data without interfering with each other’s display. I also separated the concerns by using different scripts: one dedicated to handling the live data fetched from the API and another to manage the UI interactions. This still did not work though.

Code Maintainability and Scalability:

I practiced writing clean, commenting, consistency of formats and modular code to allow maintenance and future updates. This practice also involves using version control systems like Git to manage changes and potentially collaborate efficiently across distributed teams.

Performance Optimization:

Effective load and speed of the website is important to prevent bounce rates, especially for commercial websites that rely on costumers. I tried to improve the speed and efficiency by using defer at places to help with the loading time in dedicated scripts for different pages. I also implemented lazy load that loads data as it comes into view rather than all at once, specifically for the home page that has many images. I also used simple shapes for the data visualizations to optimize D3 renderings.

I realized that the use of many images and the fact that they were not optimized (compressed and resized) made the images load take time to load upon first opening the site or when refreshing it, which affect the users experience and user engagement to a certain extent. Although I reduced text and primarily used images to enhance the narrative, headings and subheadings were still employed to reinforce the images' purpose. I incorporated alt text to provide users with basic information while the images loaded, preventing blank spaces on the site. However, this approach particularly affected mobile users, who typically have devices with less processing power than desktops. Large, unoptimized images can deplete device resources, resulting in sluggish performance and a suboptimal user experience. Additionally, the increased data consumption due to large image files is not ideal, as it can lead to higher bandwidth usage.

Responsive Design:

Firstly, responsive design practices align with international web development standards, such as those set by the World Wide Web Consortium (W3C). These standards advocate for web accessibility and usability across different platforms and devices, ensuring a wider reach and inclusivity. I ensured that my site is usable on various devices, including desktops, tablets, and smartphones through flexbox and media queries in CSS to create a seamless experience across different screen sizes and orientations. This directly contributes to a positive UX, it prevents usability issues such as unreadable text, unclickable links, or distorted layouts, which can frustrate users and drive them away. Since search engines like Google have adapted their algorithms to prioritize mobile-friendly websites in their rankings. Responsive design is integral to SEO strategies as it helps in reducing bounce rates and increasing time on site by providing a better user experience. Moreover, maintaining a single responsive site rather than separate desktop and mobile versions simplifies URL structure and eliminates duplicate content, both of which are beneficial for SEO.

Considering that is a commercial website, from a business perspective, responsive design aligns with professional practices by being cost-effective. Maintaining one website that adapts to all devices, rather than separate sites for desktop and mobile, reduces development and maintenance costs. It simplifies the web development process and decreases the likelihood of inconsistencies between different versions of a site. As new devices with different screen sizes and resolutions come to market, a responsive website can adapt to these without needing a complete redesign. This future-proofing aspect makes responsive design a sustainable choice for long-term development.
